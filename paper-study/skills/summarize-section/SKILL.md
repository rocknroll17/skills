# 섹션 5층 브리핑

사용자가 지정한 섹션을 5층으로 짧게 요약. CLAUDE.md §3·§6 준수 (400단어 이내, 헤더로 구분).

## 1. 한 줄 (TL;DR)
섹션 전체를 **한 문장**. 초심자가 이것만 봐도 "왜 이 섹션이 있는지" 알 수 있어야 함.

## 2. 기술 개요
- 주요 기여·새로움 2~3 bullet
- 핵심 알고리즘·모듈 이름
- 입력 / 출력

## 3. 상세
- 핵심 수식 1~3개를 `(§X.Y, Eq. N)` 인용
- 각 수식을 1~2문장 해설 (자세한 건 `/explain-equation`)
- 그림·표 인용 시 번호·위치 병기

## 4. 비판 (Critical lens)
- 암묵적 가정 2~3 bullet
- 약점·한계
- "만약 ___ 라면 깨질까?" edge case 1개

## 5. 선행 연구 맥락
- 어떤 이전 방법을 **확장/대체/반박**하는지
- 차별점 한 문장

## 출력 규칙

- 전체 400단어 이내.
- 모든 주장에 논문 위치 인용 `(§X.Y, p.K)` 표기.
- 결과를 `notes/0X-<섹션명>.md`에 누적 저장 권장 (사용자가 명시 요청 시에만).
- 마지막 1줄: "다음 단계 제안" (예: "Eq. 3 보고 싶으면 `/explain-equation Eq. 3`").
- 자발 Socratic 질문·Feynman 루프 금지.
