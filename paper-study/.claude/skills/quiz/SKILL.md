# Active-recall 퀴즈

주제를 받아 Bloom's taxonomy 상위 3단계로 퀴즈 **3문항** 생성.

**CLAUDE.md §11 준수**: 이 skill은 사용자가 **명시적으로 호출할 때만** 실행한다. 자발적으로 퀴즈를 꺼내지 않는다.

## 출제 규칙

1. **문항 1 [이해]**: 개념을 자기 말로 재설명하라는 open-ended 질문. 정답 키워드 3~5개 내부에 미리 정해둠.
2. **문항 2 [적용]**: 가상의 toy scenario를 주고 논문의 방법을 적용하면 어떻게 되는지 추론하게 함.
3. **문항 3 [분석]**: 논문 가정이 깨지는 edge case나 ablation 해석.

## 출력 포맷

```
### 퀴즈: <주제>

**문항 1 [이해]**
<질문>

**문항 2 [적용]**
<질문>

**문항 3 [분석]**
<질문>

(답을 한꺼번에 적어 보내주세요. 제가 한 번에 피드백.)
```

## 채점

사용자 답을 받으면 문항별로:
- **잘한 점** 1줄
- **놓친 점** 논문 인용 `(§X.Y, p.K)` 병기, 1~2줄
- **다음 제안**: "이 오답은 `/explain-equation ___` 로 복습 권함" 1줄

전체 응답 300단어 이내. 평가 설교 금지.

## 주의

- 논문에 없는 지식으로 fancy 질문 내지 말 것.
- 사용자가 "모르겠다"고 답해도 비판 금지. 힌트 1개만 주고 재시도 유도.
- 브리핑 모드 — 따뜻하되 건조하게.
